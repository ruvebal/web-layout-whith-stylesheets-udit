Selected Tools and Frameworks

BOOTSTRAP

No Node.js

Category: Standalone CSS Frameworks

What I think: Bootstrap is a classic, robust framework with prebuilt components and responsive grids. It’s reliable and fast to implement.

Pros: Easy to use, consistent design system.
Cons: Can look generic without customization.

VITE

Requires Node.js

Category: UI Kits and Component Libraries

What I think: Vite is incredibly fast for local development. I love how instant changes appear — it feels modern and minimal.

Pros: Lightning-fast HMR, minimal config.
Cons: Needs Node.js and can be tricky to set up for big projects.

REACT

Requires Node.js

Category: JavaScript Framework

What I think: React feels modular and efficient, perfect for reusable components. As a designer, I like how it pairs with libraries like Tailwind or Chakra.

Pros: Reusable UI components, large community.
Cons: Complex setup for beginners, steep learning curve.

JEKYLL

No Node.js Requires Ruby

Category: Static Site Generator

What I think: Classic static site generator, very stable and widely used in blogs and documentation.

Pros: Simple workflow, strong community.
Cons: Requires Ruby, less modern features.

ANIME.JS

No Node.js

Category: Animation and Visual Library

What I think: Anime.js is a lightweight JavaScript animation library that works directly in the browser. It’s perfect for adding interactive animations without a build step.

Pros: Easy to use, flexible, works anywhere.
Cons: Focuses only on animation; needs other tools for full UI.

HUGO

No Node.js

Category: Static Site Generator

What I think: Hugo is a fast static site generator that works without Node.js. Perfect for simple, classic sites with minimal setup.

Pros: Very fast, easy deployment.
Cons: Less modern tooling compared to Node.js frameworks.

HTML5 UP

No Node.js

Category: Templates and Open UI Kits

What I think: HTML5 UP provides free, modern HTML templates that can be used directly without any build tools. Perfect for rapid prototyping and learning HTML/CSS.

Pros: Ready-to-use templates, easy to customize, especially for beginners.
Cons: Less flexible than a full framework.

SHADCN UI

Requires Node.js

Category: Stylesheets and CSS Frameworks

What I think: I like its modern minimal style — perfect for designers who like clarity and polish.

Pros: Beautifully designed, integrates with Tailwind.
Cons: Requires setup knowledge, tied to React.

Critical Reflection

During this research, I explored several modern web development tools that have transformed how designers and developers build websites. Compared to traditional methods—where HTML and CSS were coded manually—modern tools automate repetitive processes and speed up development.

Frameworks like React and Vite illustrate a shift toward component-based design, allowing better reuse and consistency. In contrast, classic static tools like Jekyll and Hugo maintain a minimalist workflow, focusing on simplicity and direct control of code

Tools such as Bootstrap and Shadcn UI show how speed and modularity are now central priorities in modern web design.

The main trade-offs observed include:

  • Speed vs. Flexibility: Build tools are faster but depend on configuration.
  • Dependencies vs. Simplicity: Modern stacks rely on Node.js, while classic tools can run anywhere.
  • Automation vs. Control: Frameworks automate layout and logic, but can feel less manual for designers who prefer hands-on structure.

    • Overall, web design has evolved from static, handcrafted sites to highly dynamic, component-based ecosystems. Both approaches have value: classic tools prioritize creative control, while modern ones emphasize scalability and productivity. Although more modern tools exist, I created this website using HTML and CSS because I am familiar with them, and they allowed me to maintain full control over the layout and styling.
